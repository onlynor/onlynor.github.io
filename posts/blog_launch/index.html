博客搭建全流程总结托管github

本文总结了使用 Hexo 静态博客框架、Fluid 主题以及 GitHub Pages 托管服务的完整搭建流程。旨在帮助新手快速搭建起一个专业、美观、可定制的个人博客。

  • 文章来源

流程概览：从零到上线

整个搭建过程可以分为四个主要阶段：

  1. 环境准备：安装必备软件，创建 GitHub 仓库。
  2. Hexo 初始化：安装 Hexo 框架，配置站点信息。
  3. 主题与内容：安装 Fluid 主题，配置图片资源管理。
  4. 部署上线：配置 Git 部署，发布到 GitHub Pages。

阶段一：环境准备与 GitHub 仓库创建

1. 安装必备软件

这是所有操作的基础，请确保以下软件已正确安装：

软件 作用 检查命令
Git 版本控制，用于下载主题和部署到 GitHub。 git --version
Node.js Hexo 框架的运行环境。 node -v 和 npm -v

提示： 安装 Git 时，请务必配置好你的用户名和邮箱，这是 Git 提交记录的身份标识。

2. 创建 GitHub Pages 仓库

  1. 登录 GitHub，点击右上角 + 号，选择 New repository。
  2. 仓库命名规范： 必须命名为 你的用户名.github.io（例如：onlynor.github.io）。
  3. 选择 Public（公开），勾选 Add a README file（添加 README 文件）。
  4. 点击 Create repository 完成创建。

配置项 作用 建议值

description 网站描述，利于 SEO。 一个基于 Hexo 和 Fluid 主题的个人博客

timezone 时区设置，确保文章时间正确。 Asia/Shanghai
